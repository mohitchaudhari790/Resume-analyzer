# Frontend-Backend Connection on Render

## ✅ Your Project is Already Configured Correctly!

Your Django project uses **relative URLs** and **Django template tags**, which means the frontend automatically connects to whatever domain it's deployed on.

## How It Works

### 1. Django Template URLs (Already Implemented)
```html
<!-- ✅ CORRECT - What you're using -->
<a href="{% url 'dashboard' %}">Dashboard</a>
<form action="{% url 'upload_resume' %}" method="POST">

<!-- ❌ WRONG - Hardcoded URLs (you're NOT using these) -->
<a href="http://localhost:8000/dashboard/">Dashboard</a>
<form action="http://localhost:8000/upload/" method="POST">
```

**Result:**
- Local: `http://localhost:8000/dashboard/`
- Render: `https://your-app.onrender.com/dashboard/`

### 2. Static Files (Already Implemented)
```html
<!-- ✅ CORRECT - What you're using -->
<link rel="stylesheet" href="{% static 'css/style.css' %}">
<script src="{% static 'js/script.js' %}"></script>

<!-- ❌ WRONG - Hardcoded paths -->
<link rel="stylesheet" href="/static/css/style.css">
```

**Result:**
- Local: `http://localhost:8000/static/css/style.css`
- Render: `https://your-app.onrender.com/static/css/style.css`

### 3. JavaScript AJAX Calls
Your `script.js` uses relative URLs:
```javascript
// ✅ CORRECT - Relative URL
window.location.href = `/jobs/?search=${encodeURIComponent(searchTerm)}`;

// This automatically becomes:
// Local: http://localhost:8000/jobs/?search=...
// Render: https://your-app.onrender.com/jobs/?search=...
```
