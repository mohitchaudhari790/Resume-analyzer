## 📋 Step-by-Step Deployment

### Step 1: Prepare Your Repository

1. **Initialize Git (if not already done)**
   ```bash
   git init
   git add .
   git commit -m "Initial commit - Ready for Render deployment"
   ```

2. **Create GitHub Repository**
   - Go to https://github.com/new
   - Create a new repository (e.g., `ai-resume-portal`)
   - Don't initialize with README (you already have files)

3. **Push to GitHub**
   ```bash
   git remote add origin https://github.com/YOUR_USERNAME/ai-resume-portal.git
   git branch -M main
   git push -u origin main
   ```

---

### Step 2: Create Render Account

1. Go to https://render.com
2. Sign up with GitHub account
3. Authorize Render to access your repositories

---

### Step 3: Create PostgreSQL Database

1. **In Render Dashboard:**
   - Click "New +" button
   - Select "PostgreSQL"

2. **Configure Database:**
   - Name: `ai-resume-db`
   - Database: `ai_resume_db`
   - User: `ai_resume_user`
   - Region: Choose closest to your users
   - Plan: Free (or paid for production)

3. **Create Database**
   - Click "Create Database"
   - Wait for database to be created
   - **IMPORTANT:** Copy the "Internal Database URL" (you'll need this)

---

### Step 4: Create Web Service

1. **In Render Dashboard:**
   - Click "New +" button
   - Select "Web Service"

2. **Connect Repository:**
   - Select your GitHub repository
   - Click "Connect"

3. **Configure Web Service:**
   ```
   Name: ai-resume-portal
   Region: Same as database
   Branch: main
   Root Directory: (leave empty)
   Runtime: Python 3
   Build Command: ./build.sh
   Start Command: gunicorn config.wsgi:application
   Plan: Free (or paid)
   ```

4. **Add Environment Variables:**
   Click "Advanced" → "Add Environment Variable"

   Add these variables:
   ```
   SECRET_KEY = [Click "Generate" button]
   DEBUG = False
   ALLOWED_HOSTS = .onrender.com
   DATABASE_URL = [Paste Internal Database URL from Step 3]
   ADMIN_URL = secure-admin-2024-change-this
   ADMIN_ENABLED = True
   RENDER = True
   PYTHON_VERSION = 3.12.3
   ```

5. **Create Web Service**
   - Click "Create Web Service"
   - Wait for deployment (5-10 minutes)

---

### Step 5: Run Migrations

1. **After deployment completes:**
   - Go to your web service dashboard
   - Click "Shell" tab
   - Run these commands:
   ```bash
   python manage.py migrate
   python manage.py createsuperuser
   ```

2. **Create Superuser:**
   - Username: your_admin_username
   - Email: your_email@example.com
   - Password: strong_password_here

---

### Step 6: Access Your Application

1. **Your app URL:**
   ```
   https://ai-resume-portal.onrender.com
   ```

2. **Admin Panel URL:**
   ```
   https://ai-resume-portal.onrender.com/secure-admin-2024-change-this/
   ```
   (Use the ADMIN_URL you set in environment variables)

## 🔒 Security Configuration

### Admin Panel Security

**Default Admin URL:** `/admin/` (INSECURE - Don't use!)

**Your Hidden Admin URL:** `/secure-admin-2024-change-this/`

**To Change Admin URL:**
1. Go to Render Dashboard
2. Select your web service
3. Go to "Environment" tab
4. Edit `ADMIN_URL` variable
5. Change to something unique (e.g., `my-secret-admin-xyz123`)
6. Save changes
7. Render will redeploy automatically

**Access Admin:**
```
https://your-app.onrender.com/YOUR_ADMIN_URL/
```

### Environment Variables Explained

| Variable | Purpose | Example |
|----------|---------|---------|
| `SECRET_KEY` | Django secret key | Auto-generated by Render |
| `DEBUG` | Debug mode | `False` (production) |
| `ALLOWED_HOSTS` | Allowed domains | `.onrender.com` |
| `DATABASE_URL` | Database connection | From Render PostgreSQL |
| `ADMIN_URL` | Hidden admin path | `secure-admin-2024` |
| `ADMIN_ENABLED` | Enable/disable admin | `True` |
| `RENDER` | Render environment flag | `True` |

## 🔧 Troubleshooting

### Issue 1: Build Failed

**Solution:**
- Check build logs in Render dashboard
- Verify `requirements.txt` has all dependencies
- Ensure `build.sh` has execute permissions

### Issue 2: Database Connection Error

**Solution:**
- Verify `DATABASE_URL` is set correctly
- Check database is running in Render
- Ensure database and web service are in same region

### Issue 3: Static Files Not Loading

**Solution:**
- Run `python manage.py collectstatic` in Shell
- Verify WhiteNoise is in `requirements.txt`
- Check `STATIC_ROOT` in settings.py

### Issue 4: Admin Panel 404

**Solution:**
- Verify `ADMIN_URL` environment variable
- Check you're using correct URL path
- Ensure `ADMIN_ENABLED=True`

### Issue 5: 500 Internal Server Error

**Solution:**
- Check logs in Render dashboard
- Verify all environment variables are set
- Run migrations: `python manage.py migrate`
